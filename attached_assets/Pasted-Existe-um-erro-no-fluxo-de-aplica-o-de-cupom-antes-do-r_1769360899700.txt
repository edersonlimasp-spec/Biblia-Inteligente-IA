Existe um erro no fluxo de aplicação de cupom antes do redirecionamento para o Mercado Pago.

PROBLEMA ATUAL
- O usuário escolhe um plano
- Ao clicar no plano, ele é redirecionado IMEDIATAMENTE para o Mercado Pago
- Não há tempo para aplicar o cupom e recalcular o valor
- O sistema ainda exige “selecionar cupom” em vez de:
  digitar cupom → aplicar → atualizar valor → só então ir ao pagamento

COMPORTAMENTO DESEJADO (NOVO FLUXO CORRETO)

1) Tela de Plano deve funcionar assim:

ETAPA 1 — Usuário escolhe o plano  
ETAPA 2 — Usuário digita o código do cupom em um input  
ETAPA 3 — Usuário clica em botão “Aplicar cupom”  
ETAPA 4 — Front chama backend para validar cupom  
ETAPA 5 — Backend retorna:
   - valor original do plano
   - valor do desconto
   - valor final com desconto
ETAPA 6 — Front atualiza a UI mostrando:
   - Preço original (riscado, se quiser)
   - Desconto aplicado
   - Preço final
ETAPA 7 — Só depois disso, ao clicar em “Assinar / Pagar”,
          o sistema cria a preferência do Mercado Pago com O VALOR JÁ COM DESCONTO.

IMPORTANTE
- NÃO redirecionar automaticamente ao clicar no plano.
- Separar claramente:
   - Seleção do plano
   - Aplicação do cupom
   - Criação do pagamento

ALTERAÇÕES TÉCNICAS NECESSÁRIAS

FRONT-END
1) Alterar o botão do plano:
   - Em vez de redirecionar direto, apenas:
     setSelectedPlan(plan)

2) Criar estado:
   selectedPlan
   couponCode
   appliedCoupon
   originalAmount
   discountAmount
   finalAmount

3) Fluxo do cupom:
   - Botão “Aplicar cupom” chama:
     POST /api/coupons/validate { code, planId }

4) Ao receber resposta válida:
   - setOriginalAmount(...)
   - setDiscountAmount(...)
   - setFinalAmount(...)

5) Botão “Assinar agora” deve:
   - Chamar backend para criar pagamento com:
     amount = finalAmount
     couponCode
     couponId
     discountAmount

BACK-END

1) Endpoint de criação de pagamento deve:
   - Receber:
     planId
     couponCode (opcional)
   - Revalidar o cupom no backend
   - Calcular novamente:
     originalAmount
     discountAmount
     finalAmount

2) Criar preferência do Mercado Pago com:
   unit_price = finalAmount

3) Salvar no banco:
   - planId
   - originalAmount
   - discountAmount
   - finalAmount
   - couponCode (se houver)

MERCADO PAGO
- Garantir que o valor enviado para o Mercado Pago
  seja SEMPRE o valor final com desconto aplicado.
- Nunca confiar no valor do front-end.

ENTREGA
- Ajustar o fluxo para:
  plano → aplicar cupom → recalcular valor → pagar
- Remover qualquer redirecionamento automático ao clicar no plano
- Informar quais arquivos foram alterados
- Confirmar que o valor no Mercado Pago é exatamente o valor promocional

OBSERVAÇÃO
Se existir hoje algo como:
onClickPlan = () => redirectToMercadoPago(planId)

Isso deve ser removido.
O redirecionamento só pode ocorrer após:
- plano selecionado
- cupom aplicado (ou não)
- valor final calculado
