========================
1) FLUXO DO USUÁRIO
========================
A) Usuário grava um sermão/reunião (usar API de gravação já existente).
B) Após salvar a gravação:
   - Mostrar card da gravação com status: “Áudio salvo”.
   - Botão “Transcrever”.
C) Ao clicar “Transcrever”:
   - Enviar áudio para Whisper (OpenAI) via backend.
   - Receber texto e salvar no banco.
   - Abrir modal com:
       • Aba “Transcrição” (texto editável em textarea)
       • Botão “Salvar edição”
       • Botão “Gerar resumo com IA”
D) Ao clicar “Gerar resumo com IA”:
   - Chamar ChatGPT com prompt estruturado (abaixo).
   - Salvar resumo estruturado no banco.
   - Exibir em Aba “Resumo” (editável também).
E) Usuário pode editar transcrição e resumo e salvar (persistência).
F) Exportação:
   - Botão “Exportar PDF” no card e dentro do modal.
G) Compartilhar:
   - Botão “Compartilhar link” gera um link seguro (token) para leitura (com permissões).

========================
4) BACKEND ENDPOINTS
========================
Criar endpoints (Express/Node ou backend atual do projeto):

POST /api/sermons/:id/transcribe
- Recebe recordingId
- Busca audioUrl ou arquivo
- Envia para OpenAI Whisper (transcriptions)
- Salva transcriptText, transcriptStatus=done
- Retorna transcriptText

POST /api/sermons/:id/summarize
- Requer transcriptText existente
- Chama ChatGPT (Responses API) com prompt estruturado
- Salva summaryJson + summaryText
- Retorna summary

PATCH /api/sermons/:id
- Atualiza title/category/tags/speaker/transcriptText/summaryText/notesText

POST /api/sermons/:id/export/pdf
- Gera PDF server-side ou client-side (preferir client com jsPDF)
- Retorna arquivo ou dispara download

POST /api/sermons/:id/share
- Gera shareToken seguro (random 32+ chars) e shareEnabled=true
- Retorna URL do link

GET /share/sermon/:token
- Página read-only do resumo/notes (sem edição)
- Sem expor dados de outros usuários.

DELETE /api/sermons/:id
- Excluir gravação + dados associados (confirm modal)

5) INTEGRAÇÃO OPENAI (SEGURA)
========================
Usar ENV/Secrets:
- OPENAI_API_KEY

Transcrição (Whisper):
- Modelo: gpt-4o-mini-transcribe (ou whisper-1, conforme SDK disponível no projeto)
- Input: arquivo de áudio
- Retornar texto limpo

Resumo (ChatGPT):
- Modelo: gpt-4.1-mini (ou equivalente já usado no projeto)
- Temperatura baixa (0.2)
- Sempre gerar JSON + versão humana.

PROMPT DE RESUMO (usar exatamente):
“Você é um assistente de anotações cristãs. A partir da transcrição abaixo, gere um resumo estruturado e conciso.
Retorne em JSON com os campos:
- titulo_tema
- versiculo_base (se houver)
- contexto (2-4 linhas)
- pontos_principais (lista)
- ilustracoes_exemplos (lista)
- aplicacoes_praticas (lista)
- decisoes_e_desafios (lista)
- citacoes_marcantes (lista curta)
- oracao_sugerida (curta)
Depois do JSON, gere uma versão formatada em texto com títulos e bullet points.
Transcrição:
<<< {transcriptText} >>>”
