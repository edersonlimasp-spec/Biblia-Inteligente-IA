PROMPT PARA REPLIT (COPIAR E COLAR)

Objetivo:
Quero um link interno que, ao ser clicado/aberto, já inicie o fluxo de instalação do PWA automaticamente quando possível (Android/Chrome/Desktop com beforeinstallprompt). No iOS e nos casos onde o prompt não existe, deve abrir o mesmo modal com instruções.

Regras:
- Não quebrar nada do app atual.
- Manter o modal bonito já criado no Dashboard (logo + instruções).
- O comportamento deve funcionar tanto:
  1) quando o usuário clica no botão "Instalar App" do Dashboard
  2) quanto quando o usuário acessa um link interno/URL específico

Implementação solicitada:

1) Criar um link interno/rota dedicada para instalação:
   - Exemplo: /install
   - Ao entrar nessa rota, automaticamente abrir o mesmo modal de instalação já criado (sem navegar para outra página além dessa rota).
   - Essa rota deve poder ser compartilhada (ex: https://bibliainteligente.replit.app/install).

2) Auto-disparo do fluxo de instalação (Android/Chrome/Desktop):
   - Se o evento beforeinstallprompt estiver disponível e já tiver sido capturado (deferredPrompt), então:
     - abrir o modal
     - automaticamente chamar deferredPrompt.prompt() assim que o modal abrir (com pequeno delay de 300–600ms para garantir render).
   - Se o deferredPrompt NÃO estiver disponível:
     - abrir o modal e exibir mensagem "Instalação não disponível agora — use o menu ⋮ do navegador > Instalar app / Adicionar à tela inicial".
     - NÃO ficar em loop tentando.

3) iOS:
   - Detectar iOS (iPhone/iPad + Safari).
   - Em iOS, NÃO tentar prompt (não existe).
   - Ao abrir /install, abrir o modal e mostrar instruções passo a passo:
     - “Abra no Safari”
     - “Toque em Compartilhar”
     - “Adicionar à Tela de Início”
   - Se o usuário abriu dentro do navegador do WhatsApp/Instagram (in-app browser), mostrar aviso:
     - “No iPhone, abra no Safari para conseguir adicionar à tela inicial” e um botão “Abrir no Safari” (pode usar window.location.href para o mesmo link e também exibir instrução de copiar/colar se necessário).

4) Unificar o comportamento:
   - Criar uma função única openInstallModal({autoPrompt: boolean}) usada pelo botão do Dashboard e pela rota /install.
   - Botão do Dashboard: chama openInstallModal({autoPrompt:false}) (ou true, se for desejado).
   - Rota /install: chama openInstallModal({autoPrompt:true}) para tentar iniciar instalação automaticamente.

5) Ajuste do evento beforeinstallprompt:
   - Garantir que o app capture beforeinstallprompt uma única vez, armazene deferredPrompt em um store/estado global (ou contexto) e atualize o UI.
   - Exemplo: window.__deferredPrompt = e; ou store React (zustand/context).
   - Lembrar de e.preventDefault().

6) Pós-ação:
   - Quando userChoice resultar accepted/dismissed, limpar deferredPrompt e atualizar o modal:
     - accepted: mostrar “Instalação iniciada com sucesso”
     - dismissed: manter instruções e permitir tentar novamente via botão “Instalar Agora”.

7) Testes mínimos:
   - Android Chrome: abrir /install => modal abre e diálogo de instalação aparece automaticamente (se elegível).
   - iPhone Safari: abrir /install => modal abre com instruções.
   - WhatsApp/Instagram iOS: abrir /install => modal abre e avisa para abrir no Safari.
   - Se não estiver elegível para instalação (já instalado, ou critérios não atendidos): mostrar instrução e não dar erro.
