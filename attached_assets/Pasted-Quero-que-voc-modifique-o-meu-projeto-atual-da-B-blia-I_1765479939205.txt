Quero que você modifique o meu projeto atual da “Bíblia Inteligente IA” acrescentando um módulo completo de gravação de áudio para sermões e mensagens.
⚠ Importante:

Não remova nem quebre nenhuma funcionalidade já existente do app.

Mantenha o layout, rotas e estilos atuais, apenas adicionando esse novo módulo.

O app deve continuar funcionando normalmente em desktop e principalmente em celular.

Objetivo do novo módulo

Criar uma área chamada algo como "Gravações" ou "Sermões Gravados" em que o usuário possa:

Gravar áudios (sermões, palavras em cultos, devocionais, etc.).

Reproduzir essas gravações posteriormente.

Salvar e organizar cada gravação (com título, data/hora, duração).

Compartilhar as gravações com outras pessoas:

Por WhatsApp (link).

Por e-mail (link).

E, se possível, via compartilhamento nativo do navegador (navigator.share).

1. Estrutura do módulo de gravação

Crie um novo módulo/página no app (por exemplo rota /gravador ou /gravacoes) e adicione um botão/card no menu principal/dashboard para acessar esse módulo.
Esse módulo deve conter:

Área principal do gravador:

Botão "Iniciar gravação".

Durante a gravação, mostrar:

Tempo decorrido (em segundos/minutos).

Indicação visual de que está gravando (por exemplo, um ponto vermelho, texto “Gravando...” ou animação simples).

Botão "Parar gravação".

Após parar a gravação, permitir:

Reproduzir o áudio.

Salvar a gravação com:

Título (campo texto obrigatório, exemplo: “Sermão de Domingo – 12/12”).

Data e hora automáticas (geradas pelo sistema).

Duração do áudio.

Lista de gravações salvas:

Exibir uma lista com todas as gravações já salvas.

Para cada item, mostrar:

Título.

Data/hora.

Duração aproximada.

Para cada gravação, adicionar botões:

Play/Pause (ou ícone de reprodução).

Compartilhar.

Excluir (com uma pequena confirmação para não apagar por engano).

2. Implementação técnica da gravação

Use a MediaRecorder API (ou solução equivalente no navegador) para capturar o áudio do microfone do usuário.

Configurações:

Priorize alta qualidade dentro do razoável (por exemplo, sample rate comum de 44.1kHz ou 48kHz, e formato como audio/webm ou audio/ogg, conforme suporte do navegador).

Trate os casos em que o navegador não suporta a API de gravação:

Mostrar uma mensagem amigável ao usuário:
“Seu navegador não suporta a gravação de áudio. Tente atualizar ou usar outro navegador.”

Trate também:

Permissão negada para microfone: mostrar mensagem clara e instrução para o usuário liberar o microfone.

3. Armazenamento dos áudios

Quero duas camadas de armazenamento:

Local (no navegador):

Use IndexedDB ou outra solução persistente no cliente para guardar:

O blob do áudio gravado.

Metadados (título, data/hora, duração, id da gravação).

Isso garante que a gravação continue disponível no app do usuário mesmo se ele atualizar a página.

Servidor (opcional, mas desejável):

Se possível, crie uma API simples no backend (Node/Express) para:

Receber upload de um áudio.

Salvar o arquivo em uma pasta tipo /recordings.

Servir esse arquivo por uma URL pública.

Também criar rotas API como:

POST /api/recordings – para salvar uma gravação (metadados e arquivo).

GET /api/recordings – para listar as gravações do usuário.

DELETE /api/recordings/:id – para excluir.

Importante:

Se isso for complexo demais para o contexto atual do meu projeto, priorize o funcionamento 100% local no navegador utilizando IndexedDB, mas mantenha o código organizado para no futuro ser fácil integrar com backend.

4. Compartilhamento das gravações

Implemente uma área de compartilhamento em cada gravação:

Se existir URL de servidor para o arquivo:

Use essa URL para compartilhar.

Se só existir localmente:

Gere uma URL temporária com URL.createObjectURL(blob).

Para o botão Compartilhar:

Tente usar a Web Share API (navigator.share) quando disponível, passando:

Título.

Texto (por exemplo: “Ouça este sermão gravado no app Bíblia Inteligente IA”).

URL da gravação (quando disponível).

Quando navigator.share não estiver disponível:

Exiba:

Um botão/link para WhatsApp Web com URL no formato:
https://wa.me/?text=... incluindo o link ou instruções.

Um botão/link para e-mail usando mailto: com:

Assunto: “Sermão gravado – Bíblia Inteligente IA”

Corpo do e-mail com uma mensagem padrão e, se possível, o link da gravação.

Caso o áudio não possa ser enviado diretamente, explique na interface que o usuário pode baixar o arquivo e anexar manualmente.

5. Interface e UX (focado em celular)

O layout deve ser responsivo e bem adaptado para smartphones.

Organizar o módulo em cards ou seções claras:

Card do gravador (no topo).

Lista de gravações logo abaixo.

Deixar os botões grandes o bastante para tocar com o dedo.

Manter a identidade visual já existente do app Bíblia Inteligente IA (cores, fontes, estilo).

6. Organização do código

Não quebre nada do código atual.

Separar o código do gravador em:

Um arquivo JS próprio (por exemplo recorder.js, sermoes.js ou similar).

Componentes ou funções bem documentadas.

Adicionar comentários no código explicando:

Como iniciar a gravação.

Como parar e salvar.

Como funciona o armazenamento.

Como funciona o compartilhamento.

Após implementar:

Testar no navegador do Replit e também no modo móvel (inspecionar, modo device).

Confirmar que:

A gravação funciona (inicia e para).
