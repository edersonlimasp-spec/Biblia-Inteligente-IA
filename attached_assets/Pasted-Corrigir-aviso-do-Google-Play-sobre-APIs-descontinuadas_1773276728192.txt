Corrigir aviso do Google Play sobre APIs descontinuadas de Edge-to-Edge no Android 15, sem quebrar o aplicativo.

Contexto do problema:
O Google Play informou que o app usa APIs/parâmetros descontinuados para exibição de ponta a ponta no Android 15. As APIs citadas são:

- android.view.Window.setNavigationBarColor
- android.view.Window.setStatusBarColor
- android.view.Window.getStatusBarColor

Os pontos citados no relatório incluem:
- J.L.d
- J.L.e
- com.google.androidbrowserhelper.trusted.WebViewFallbackActivity.onCreate

Objetivo:
Migrar a implementação para o padrão atual recomendado no Android 15+, removendo dependências de APIs descontinuadas, preservando o funcionamento do app e evitando regressões visuais.

Tarefas:
1. Fazer varredura completa no projeto Android para localizar qualquer uso direto ou indireto de:
   - setNavigationBarColor
   - setStatusBarColor
   - getStatusBarColor
   - window status/navigation bar color legacy handling
   - flags antigas de window/system UI

2. Não fazer alteração agressiva que quebre a UI.
   Aplicar correções incrementais e seguras.

3. Na MainActivity (ou Activity principal), migrar para Edge-to-Edge moderno e compatível com versões anteriores:
   - usar enableEdgeToEdge() se estiver em Kotlin
   - ou EdgeToEdge.enable(this) se estiver em Java
   - aplicar WindowInsets corretamente no conteúdo principal

4. Remover lógica manual legada de colorização de status bar e navigation bar quando ela depender dessas APIs descontinuadas no Android 15.
   Em vez disso:
   - usar edge-to-edge com tratamento de insets
   - usar aparência clara/escura das barras com WindowInsetsControllerCompat, se necessário
   - manter comportamento visual equivalente sempre que possível

5. Verificar temas/styles.xml/themes.xml:
   - remover ou ajustar configurações antigas que forcem comportamento legado de barras do sistema
   - manter compatibilidade visual
   - evitar uso de statusBarColor/navigationBarColor como solução principal para Android 15+

6. Investigar dependências que possam estar causando o aviso, especialmente:
   - com.google.androidbrowserhelper
   - Trusted Web Activity / WebViewFallbackActivity
   - bubblewrap / wrappers relacionados

7. Se o uso vier de dependência antiga:
   - atualizar a dependência para a versão estável mais recente compatível com o projeto
   - se houver breaking change, adaptar o código
   - não atualizar outras bibliotecas desnecessariamente

8. Confirmar se o problema vem do android-browser-helper / WebViewFallbackActivity.
   Se vier dessa biblioteca:
   - atualizar a dependência

   - preservar o funcionamento da TWA/PWA/WebView fallback

9. Testar e validar:
   - app inicia sem crash

   - conteúdo inferior não fica atrás da navigation bar
   - telas com rolagem continuam normais
   - splash screen continua funcionando
   - trusted web activity continua abrindo normalmente

10. No final, me mostrar:
   - arquivos alterados
   - dependências atualizadas
   - o que foi removido
   - por que isso corrige o aviso
   - se ainda existir algo vindo de biblioteca de terceiros

- Não quebrar build
- Não remover funcionalidades do app
- Não alterar lógica de negócio
- Manter compatibilidade com Android 10 até Android 15+
- Preferir solução oficial AndroidX moderna
